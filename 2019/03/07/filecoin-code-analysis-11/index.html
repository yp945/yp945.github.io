filecoin技术架构分析十一：filecoin源码分析之内部接口层api包分析

发表于 2019-03-07 更新于 2019-07-05 分类于 FileCoin 7.1k 14 分钟

我是先河系统CTO杨尉，欢迎大加关注的的Github: waynewyang，本文是filecoin技术架构分析系列文章第十一章源码分析之内部接口层api包分析。

api包提供内部接口,供协议层、command/REST使用

较大程度依赖node包

api

api的接口定义

如下所示，包含了一系列子接口

api的接口实现

actor的接口定义

actor的接口实现

address

  • 提供功能
    • 地址显示方法
    • 地址查找方法
    • 创建地址方法
    • 导出地址方法
    • 导入地址方法

client

  • 提供如下功能
    • 查询piece数据（DAG格式）
    • 导入数据（相当于ipfs add）
    • 列出所有订单
    • 支付
    • 发起存储交易
    • 查询存储交易

config

  • 提供功能
    • Get配置
    • Set配置

daemon

  • 提供功能
    • 启动进程相关
    • 具体的业务启动逻辑会调用到node包

dag

  • 提供功能
    • dag查询功能
    • 类似ipfs block get

id

  • 提供功能
    • ID详细信息
    • 如多地址、协议版本、导出公钥等

log

  • 提供日志功能

miner

  • 创建矿工

mining

  • 挖矿控制
    • 启动
    • 停止

ping

  • 提供ping接口

retrieval_client

  • 提供检索接口

swarm

  • 提供节点连接功能
    • 显示连接节点
    • 连接节点
    • 查找节点
